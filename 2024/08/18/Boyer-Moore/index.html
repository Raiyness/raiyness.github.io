What is Boyer-Moore String-search Algorithm

In computer science, the Boyer–Moore string-search algorithm is an efficient string-searching algorithm that is the standard benchmark for practical string-search literature.
在计算机科学中，Boyer-Moore 字符串搜索算法是一种高效的字符串搜索算法，它是实际字符串搜索文献的标准基准。

The algorithm preprocesses the string being searched for (the pattern), but not the string being searched in (the text). It is thus well-suited for applications in which the pattern is much shorter than the text or where it persists across multiple searches.
算法对被搜索的字符串（模式）进行预处理，但不对被搜索的字符串（文本）进行预处理。因此，它非常适用于模式比文本要短得多或在多次搜索中保持不变的应用场景。

Definitions

  • T denotes the input text to be searched. Its length is n.
  • P denotes the string to be searched for, called the pattern. Its length is m.
  • S[i] denotes the character at index i of string S, counting from 1.
  • S[i..j] denotes the substring of string S starting at index i and ending at j, inclusive.
  • A prefix of S is a substring S[1..i] for some i in range [1, l], where l is the length of S.
  • A suffix of S is a substring S[i..l] for some i in range [1, l], where l is the length of S.
  • An alignment of P to T is an index k in T such that the last character of P is aligned with index k of T.
  • A match or occurrence of P occurs at an alignment k if P is equivalent to T[(k-m+1)..k].
\begin{algorithm}

\caption{Monotonic Stack for Finding the Next Greater Element}

\begin{algorithmic}

\Procedure{NextGreaterElement}{numsnumsnums}

\State $stack \gets \emptyset$ \Comment{Initialize an empty stack}

\State $result \gets$ array of size $|nums|$ filled with $-1$

\Comment{Initialize result array with -1}

\For{$i \gets 0$ \To $|nums| - 1$}

  \While{$stack$ is not empty and $nums[stack.top()] < nums[i]$}

    \State $index \gets stack.pop()$

    \State $result[index] \gets nums[i]$

  \EndWhile

  \State $stack.push(i)$

\EndFor

\State \Return $result$

\EndProcedure
\end{algorithmic}
\end{algorithm}

Description

The Boyer–Moore algorithm searches for occurrences of PPP in TTT by performing explicit character comparisons at different alignments. Instead of a brute-force search of all alignments (of which there are n−m+1n-m+1n−m+1), Boyer–Moore uses information gained by preprocessing PPP to skip as many alignments as possible.
Boyer-Moore 算法通过在不同的对齐位置执行显式字符比较来搜索 TTT 中 PPP 的出现。而不是对所有可能的对齐位置进行蛮力搜索（总共有 n−m+1n-m+1n−m+1 个），Boyer-Moore 利用预处理 PPP 所得到的信息，尽可能地跳过多余的对齐位置。

The key insight in this algorithm is that if the end of the pattern is compared to the text, then jumps along the text can be made rather than checking every character of the text. The reason that this works is that in lining up the pattern against the text, the last character of the pattern is compared to the character in the text. If the characters do not match, there is no need to continue searching backwards along the text. If the character in the text does not match any of the characters in the pattern, then the next character in the text to check is located m characters farther along the text, where m is the length of the pattern. If the character in the text is in the pattern, then a partial shift of the pattern along the text is done to line up along the matching character and the process is repeated. Jumping along the text to make comparisons rather than checking every character in the text decreases the number of comparisons that have to be made, which is the key to the efficiency of the algorithm.
该算法的关键洞察是，如果将模式的末尾与文本进行比较，那么可以跳过文本中的一些字符而不必检查每个字符。这样做的原因是，在将模式与文本对齐时，会将模式的最后一个字符与文本中的字符进行比较。如果字符不匹配，则无需继续向后搜索文本。如果文本中的字符与模式中任何一个字符都不匹配，则需要在文本上再向前移动 m 个位置来检查下一个要比较的字符，其中 m 是模式长度。如果文本中的字符在模式中存在，则会对模式进行部分移位以使其与匹配到的字符对齐，并重复此过程。通过跳过一些文字来进行比较而不是检查每个文字，减少了必须执行的比较次数，这就是该算法高效性能背后关键所在。

More formally, the algorithm begins at alignment k=mk=mk=m, so the start of PPP is aligned with the start of TTT. Characters in PPP and TTT are then compared starting at index mmm in PPP and kkk in TTT, moving backward. The strings are matched from the end of PPP to the start of PPP. The comparisons continue until either the beginning of PPP is reached (which means there is a match) or a mismatch occurs upon which the alignment is shifted forward (to the right) according to the maximum value permitted by a number of rules. The comparisons are performed again at the new alignment, and the process repeats until the alignment is shifted past the end of TTT, which means no further matches will be found.
更正式地说，算法从对齐 k=mk=mk=m 开始，因此 PPP 的开头与 TTT 的开头对齐。然后从 PPP 中的索引 mmm 和 TTT 中的索引 kkk 开始向后比较字符。字符串从 PPP 的末尾到 PPP 的开头进行匹配。比较将继续，直到达到 PPP 的开头（表示有匹配）或发生不匹配为止，在这种情况下，根据一些规则允许的最大值向前（向右）移动对齐。在新位置再次执行比较，并重复该过程，直到对齐超出了 TTT 的末尾，这意味着不会找到进一步的匹配。

The shift rules are implemented as constant-time table lookups, using tables generated during the preprocessing of PPP.
实施移位规则时，使用在预处理 PPP 期间生成的常数时间表查找。

Shift Rules

A shift is calculated by applying two rules: the bad-character rule and the good-suffix rule. The actual shifting offset is the maximum of the shifts calculated by these rules.
通过应用两个规则来计算移动偏移量：坏字符规则和好后缀规则。实际的移动偏移量是由这些规则计算出的最大值。

The Bad-character Rule

Description

The bad-character rule considers the character in TTT at which the comparison process failed (assuming such a failure occurred).
坏字符规则考虑了在比较过程中失败的 T 中的字符（假设发生了这样的失败）。

  • The next occurrence of that character to the left in PPP is found, and a shift which brings that occurrence in line with the mismatched occurrence in TTT is proposed.
    找到 P 中该字符左侧的下一个出现位置，并提出一种移动，使得该位置与 T 中不匹配的位置对齐。
  • If the mismatched character does not occur to the left in PPP, a shift is proposed that moves the entirety of PPP past the point of mismatch.
    如果不匹配的字符在 P 中不存在左侧，则提议进行整体移动，将 P 完全移到不匹配点之后。

Preprocessing

Methods vary on the exact form the table for the bad-character rule should take, but a simple constant-time lookup solution is as follows: create a 2D table which is indexed first by the index of the character ccc in the alphabet and second by the index iii in the pattern. This lookup will return the occurrence of ccc in PPP with the next-highest index j<ij<ij<i⁠ or −1-1−1 if there is no such occurrence. The proposed shift will then be ⁠i−ji-ji−j⁠, with ⁠O(1)O(1)O(1)⁠ lookup time and ⁠O(km)O(km)O(km) space, assuming a finite alphabet of length kkk.
在坏字符规则中，表格的具体形式可能有所不同，但一个简单的常数时间查找解决方案如下：创建一个二维表格，首先按字母表中字符 ccc 的索引进行索引，然后按模式中索引 iii 进行第二次索引。这个查找将返回 PPP 中字符 ccc 在下一个最高索引 j<ij<ij<i⁠处的出现位置，如果没有这样的出现，则返回 -1。提议的移动量将是⁠i−ji-ji−j⁠，具有⁠ O(1)O(1)O(1) ⁠查找时间和 ⁠ O(km)O(km)O(km) 空间复杂度（假设字母表长度为 kkk）。

The Good-suffix Rule

Description

The good-suffix rule is markedly more complex in both concept and implementation than the bad-character rule. Like the bad-character rule, it also exploits the algorithm’s feature of comparisons beginning at the end of the pattern and proceeding towards the pattern’s start. It can be described as follows:
好后缀规则在概念和实现上比坏字符规则更加复杂。与坏字符规则类似，它也利用了算法从模式的末尾开始比较并向模式的起始位置前进的特点。可以描述为：

Suppose for a given alignment of PPP and TTT, a substring ttt of TTT matches a suffix of PPP and suppose ttt is the largest such substring for the given alignment.
假设对于给定的 PPP 和 TTT 的对齐，字符串 ttt 是 TTT 的一个子串，与 PPP 的后缀匹配，并且假设在给定的对齐中， ttt 是最大的这样子串。

  1. Then find, if it exists, the right-most copy t′t't′ of ttt in PPP such that ttt is not a suffix of PPP and the character to the left of t′t't′ in PPP differs from the character to the left of t′t't′ in PPP. Shift PPP to the right so that substring t′t't′ in PPP aligns with substring ttt in TTT.
    然后，在 PPP 中找到，如果存在的话，最右边的 ttt 的副本 t′t't′，使得 ttt 不是 PPP 的后缀，并且在 PPP 中与 t′t't′ 左边的字符不同。将字符串 t′t't′ 在 PPP 中向右移动，使其与字符串 ttt 在 TTT 中对齐。
  2. If t′t't′ does not exist, then shift the left end of PPP to the right by the least amount (past the left end of ttt in TTT) so that a prefix of the shifted pattern matches a suffix of ttt in TTT. This includes cases where ttt is an exact match of PPP.
    如果 t′t't′ 不存在，则将 PPP 的左端向右移动最少的量（超过 TTT 中 ttt 的左端）以使得移动后的模式前缀与 TTT 中 ttt 的后缀匹配。这包括了 ttt 与 PPP 完全匹配的情况。
  3. If no such shift is possible, then shift PPP by mmm (length of PPP) places to the right.
    如果没有这样的移位可能，则将 PPP 向右移动 mmm（PPP 的长度）个位置。

Preprocessing

The good-suffix rule requires two tables: one for use in the general case (where a copy t′t^{\prime}t′ is found), and another for use when the general case returns no meaningful result. These tables will be designated LLL and HHH respectively. Their definitions are as follows:
好后缀规则需要两个表格：一个用于一般情况（当找到副本 t′t^{\prime}t′ 时），另一个用于一般情况返回无意义结果时使用。这些表将分别被指定为 LLL 和 HHH。它们的定义如下：

  • For each i,L[i]i, L[i]i,L[i] is the largest position less than mmm such that string P[i..m]P[i . . m]P[i..m] matches a suffix of P[1..L[i]]P[1 . . L[i]]P[1..L[i]] and such that the character preceding that suffix is not equal to P[i−1].L[i]P[i-1] . L[i]P[i−1].L[i] is defined to be zero if there is no position satisfying the condition.
    对于每个 iii，L[i]L[i]L[i] 是小于 mmm 的最大位置，使得字符串 P[i..m]P[i . . m]P[i..m] 与 P[1..L[i]]P[1 . . L[i]]P[1..L[i]] 的后缀匹配，并且该后缀前面的字符不等于 P[i−1]P[i-1]P[i−1]。如果没有满足条件的位置，则定义 L[i]L[i]L[i] 为零。
  • Let H[i]H[i]H[i] denote the length of the largest suffix of P[i..m]P[i . . m]P[i..m] that is also a prefix of PPP, if one exists. If none exists, let H[i]H[i]H[i] be zero.
    让 H[i]H[i]H[i] 表示 P[i..m]P[i . . m]P[i..m] 的最长后缀，如果它也是 PPP 的前缀。如果不存在这样的后缀，则令 H[i]H[i]H[i] 为零。

Both of these tables are constructible in O(m)O(m)O(m) time and use O(m)O(m)O(m) space. The alignment shift for index iii in PPP is given by m−L[i]m-L[i]m−L[i] or m−H[i]m-H[i]m−H[i]. HHH should only be used if L[i]L[i]L[i] is zero or a match has been found.
这两个表格都可以在 O(m)O(m)O(m) 的时间内构建，并且使用 O(m)O(m)O(m) 的空间。对于 PPP 中索引 iii 的对齐偏移由 m−L[i]m-L[i]m−L[i] 或 m−H[i]m-H[i]m−H[i] 给出。只有当 L[i]L[i]L[i] 为零或找到匹配时才应该使用 HHH。
